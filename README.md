# art-gallery-backend

Flask ベースのバックエンド API サーバーです。アートギャラリーの作品管理と注文処理を提供します。

## 概要

このリポジトリは、マルチレポ構成におけるバックエンドアプリケーションの実装コードを管理します。インフラ関連の設定（Dockerfile、requirements.txt 等）は `art-gallery-release-tools` リポジトリで一元管理されています。

- **[セキュリティガイド](docs/SECURITY.md)**:

## 特徴

- **クリーンアーキテクチャ**: ドメイン層、リポジトリ層、サービス層、ルーティング層に明確に分離された設計。
- **テスト駆動開発**: ユニットテストと結合テストを網羅的に実装。
- **型安全性**: 型ヒントを活用したコード品質の向上。

## ディレクトリ構成

- `app.py`: アプリケーションエントリーポイント
- `config/`: 設定管理モジュール（`art-gallery-secrets-api` による機密情報の取得を含む）
- `domain/`: ドメインモデル層（エンティティ定義）
- `repositories/`: データアクセス層（データベース接続とリポジトリパターン）
- `services/`: ビジネスロジック層
- `routes/`: HTTP ルーティング層
- `tests/`: テストコード（ユニットテスト、結合テスト）

## 開発環境のセットアップ

詳細は `art-gallery-release-tools` リポジトリのドキュメントを参照してください。

## CI/CD

このリポジトリの CI ワークフロー（`.github/workflows/ci.yml`）は、コード品質チェック（テスト、Linter）のみを実行します。Docker イメージのビルドやデプロイは `art-gallery-release-tools` リポジトリから手動で実行されます。
